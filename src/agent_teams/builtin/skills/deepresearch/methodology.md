# 深度研究方法：五阶段流程

## 概述

本文档详细介绍了开展深度研究的方法。这五个阶段代表了一种全面的方法，用于收集、验证和综合来自多个来源的信息。

---

## 第一阶段：主题分解

**目标：** 明确研究范围和成功标准

**活动：**
1. 将问题分解为核心组成部分
2. 识别利益相关者的观点
3. 明确范围边界（包含/排除哪些内容）
4. 建立成功标准
5. 列出需要验证的关键假设

**超强思维应用：** 在确定研究范围之前，运用扩展推理来探索问题的多种框架。

**输出：**包含研究范围的结构化范围文件

---

## 第二阶段：研究路径图规划

**目标：**制定智能研究路线图

**活动：**
1. 确定一手资料和二手资料
2. 绘制知识依赖关系图（首先必须理解什么）
3. 创建多样的搜索查询策略
4. 计划三角测量法
5. 估算每个阶段所需的时间
6. 定义质量标准

**思维导图：** 分解出多个潜在的研究路径，然后收敛到最优策略。

**输出：** 包含优先调查路径的研究计划

---

## 第三阶段：多轮迭代并行搜索

**目标：**利用并行执行，系统地从多个来源收集信息，以实现最高速度
**关键：单消息使用多个工具调用并行执行所有搜索**

### 查询分解策略

在开展搜索之前，将研究问题分解为 5-10 个独立的搜索方案：

1. **核心主题（语义搜索）** - 基于意义的主要概念探索
2. **技术细节（关键词搜索）** - 具体术语、API、实现方式
3. **近期发展动态（按日期筛选）** - 2024-2025 年的新变化
4. **学术资源（特定领域）** - 论文、研究、正式分析
5. **其他视角（比较）** - 竞争性方法、批评意见
6. **统计/数据来源** - 量化证据、指标、基准
7. **行业分析** - 商业应用、市场趋势
8. **关键分析/局限性** - 已知问题、失效模式、极端情况

### 并行执行协议
**步骤 1：广度搜索（建立全局认知）**

**重要提示：请使用正确的工具和参数，以避免出错**

每次研究会话选择一种检索方法：

**选项 A：使用 WebSearch（内置，无需 MCP）**
- 使用简单查询字符串进行标准网络搜索
- 参数：`query`（必填），可选 `num_results`、`allowed_domains`、`blocked_domains`
- 返回结构化命中结果；直接使用命中里的 URL 作为来源，不要再从纯文本摘要里猜链接
- 例如：`WebSearch(query="quantum computing 2025")`

**方案B：使用MCP（如有，功能更强大）**
- 高级语义+关键词搜索
- 工具名称：`mcp_search`
- 参数：`query`（必需），`num_results`
- 例如：`mcp_search(query="量子计算", num_results=10)`

**切勿混用参数样式** - 这会导致"Invalid tool parameters"错误。

** 步骤 2：Spawn parallel deep-dive agents**

使用任务工具和通用代理（3-5 agents）执行以下任务：
- 学术论文分析（PDF，详细提取）
- 文档深度解析（技术规范、API 文档）
- 代码库分析（代码示例、实现）
- 专业领域研究（需要多步骤调查）

**并行执行示例（使用 WebSearch）：**

```
[一条消息包含多个工具调用]
- websearch(query="2025 年量子计算现状")
- websearch(query="量子计算的局限性和挑战")
- websearch(query="2024-2025 年量子计算商业应用")
- websearch(query="量子计算与经典计算的比较")
- websearch(query="量子纠错研究")
- Task(subagent_type="General", description="分析量子计算论文", prompt="深入研究2024-2025年的量子计算学术论文，提取关键发现和方法论")
- Task(subagent_type="General", description="行业分析", prompt="分析量子计算行业报告和市场数据，识别商业应用")
```

**并行执行示例（使用MCP，如果可用）：**
```
[一条消息包含多个工具调用]
- mcp_search(query="量子计算的最新进展", type="auto", num_results=10, start_published_date="2024-01-01")
- mcp_search(query="量子计算的局限性", type="keyword", num_results=10)
- mcp_search(query="量子计算商业", type="auto", num_results=10, start_published_date="2024-01-01")
- mcp_search(query="量子纠错", type="auto", num_results=10, include_domains=["arxiv.org"])
- Task(subagent_type="General", description="学术分析", prompt="分析量子计算学术论文")
```
**步骤 3：收集和整理结果**

结果出来后：
1. 提取关键段落及其来源元数据（标题、URL、日期、可信度）
2. 追踪出现的信息缺口
3. 利用有前景的线索进行更多有针对性的搜索
4. 保持信息来源多样性（包括学术文献、行业文献、新闻报道和技术文档）
5. 监控质量阈值（见下方 FFS 模式）

### 首次完成搜索 (FFS) 模式

**基于质量阈值的自适应补全：**

**质量关卡：**当达到第一个阈值时，进入第四阶段：
- **快速模式：** 10 个以上信息来源，平均可信度 >60/100 或 2 分钟内完成
- **深度模式：** 20 个以上信息源，平均可信度 >70/100 或 10 分钟后
- **专家级模式：** 30 个以上信息源，平均可信度 >75/100 或 15 分钟后

继续进行背景调查：
- 如果提前达到阈值，则继续在后台进行剩余的并行搜索。
- 第五阶段（综合分析）中使用的其他数据源，用于增强深度和多样性
- 既能快速推进，又不失严谨性

### 质量标准

**来源多样性要求：**
- 至少 3 种信息来源类型（学术文献、行业文献、新闻报道、技术文档）
- 时间多样性（包含 2024-2025 年的最新数据以及更早期的基础数据）
- 视角多样性（支持者 + 批评者 + 中立分析）
- 地域多样性（不仅限于本国来源）

**信誉度跟踪：**
- 使用 source_evaluator.py 为每个资源打分，分值范围为 0-100。
- 将可信度较低的信息源（<40）标记为需要进一步核实的信息来源。
- 核心论点应优先采用高可信度来源（>80）。

**技术：**
- 使用 websearch 获取最新信息（主要工具）
- 使用 webfetch 对特定来源进行深入挖掘（次要工具）
- 使用 Exa search 搜索（通过 websearch) 进行语义探索
- 使用 Grep/Read 查看本地文档
- 执行计算分析代码（需要时）
- 使用任务工具spawn parallel retrieval agents (3-5 agents)

**输出：** 包含来源追踪、可信度评分和覆盖范围图的组织化信息库

---


## 第四阶段：交叉引用验证
**目标：**验证来自多个独立来源的信息
-**活动：**
1. 识别需要核实的说法
2. 交叉核对三个以上来源的事实
3. 旗帜上的矛盾或不确定性
4. 评估信息来源的可信度
5. 注意共识领域与争议领域
6. 每项索赔的文件核实状态

质量标准：
- 核心论点必须有3个以上独立来源。
- 标记任何单一来源信息
- 注意信息的时效性
识别潜在偏见

**输出：** 已验证的事实基础及其置信度

### 数据冲突处理
当不同来源数据冲突时：
1. 列出所有来源数据
2. 分析差异原因（统计口径、时间范围、定义差异）
3. 选择最权威/最新来源作为主数据
4. 在报告中明确说明数据范围和不确定性

### 幻觉风险防控

#### 高风险场景识别
- 具体数字（金额、比例、日期）
- 人名、公司名、产品名
- 因果关系陈述
- 预测性结论

#### 防控措施
1. **源头验证**：所有关键数据必须有明确来源
2. **交叉验证**：重要数据至少2个独立来源
3. **不确定性标注**：对单一来源或推测性内容明确标注
4. **留白原则**：找不到可靠数据时，明确标注为"信息缺口"而非编造

---

## 第五阶段：报告生成

**目标：**提供专业、可操作的研究成果

**活动：**
1. 结构清晰、层级分明的报告。
2. 撰写摘要
3. 制定详细章节
4. 创建可视化图表（表格、图表）
5. 编制完整的参考文献
6. 添加方法论附录

**输出：**完整的、可供使用的研究报告

---

## 高级功能

### 思维图推理

不要采用线性思维，而是要探索多种推理路径：
- 并行探索其他框架
- 追查可能相关的旁支线索
- 合并来自不同分支的见解
- 根据新信息进行回溯和修改

### 并行代理部署

使用任务工具生成以下子代理：
- 并行源检索
- 独立验证路径
- 竞争假设评估
- 专业领域分析

### 自适应深度控制

根据以下条件自动调整研究深度：
信息复杂性
- 资源可用性
时间限制
置信水平

### 引文智能

智能引用管理：
- 追踪每一项索赔的来源
- 链接到原始来源
- 评估信息来源的可信度
处理相互冲突的信息来源
生成规范的参考文献
