Create one or more run-scoped delegated task contracts.

Usage:
- Each task entry must include a clear `objective`; `title` is optional.
- Use this tool to define work as a durable task contract.
- Write objectives as stable outcomes and constraints, not as one-off execution prompts.
- To create a dynamic DAG, set `role_id` and `orchestration_node_id` on each node and express edges with `depends_on_node_ids`.
- A task with `role_id` is pre-bound and queued; ready nodes run automatically after the current Coordinator turn, respecting `depends_on_node_ids` and `depends_on_task_ids`.
- Use `orch_dispatch_task` later only when a task was created without `role_id` or when you need to execute one task immediately.
- The result returns task identifiers; use those ids with `orch_update_task`, `orch_list_delegated_tasks`, or `orch_dispatch_task`.
