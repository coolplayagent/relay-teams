# aipptx 技能使用指南

### 页面生成
逐页生成 SVG，16:9 比例，分两个阶段执行。

#### 【阶段一：低分辨率完整稿】

viewBox="0 0 640 360"，文件命名 `page_N_draft.svg`

**目标**：在低分辨率下生成内容完整、布局合理、可独立阅读的页面。不是骨架占位，而是"缩小版的成品"。

**内容要求**：
- 所有文字必须是真实内容，禁止使用占位矩形代替文字
- content.md 中该页的全部信息点都必须体现，不遗漏
- 标题、正文、表格数据、标签文字等全部填入真实文本
- 数据/对比/趋势 → 绘制实际图表（柱状、折线等）而非预留空白区域
- 步骤/流程 → 绘制完整节点+连线+文字标注
- 并列信息 → 分栏/卡片化并填入内容

**布局要求**：
- 明确划分标题区、主体区、辅助区、结论区，版面饱满均匀
- 文字不超界、不重叠，重要模块尺寸大于次级模块
- 遵循风格参考文档的配色、字体、布局结构

**验收标准**：
- 页面在 640×360 尺寸下内容完整可读
- 不存在空白占位块或无文字的纯色矩形（装饰元素除外）
- 即使不经过阶段二，也能传达完整信息

#### 【阶段二：高分辨率精装修】

viewBox="0 0 1280 720"，文件命名 `page_N.svg`

**目标**：在阶段一的完整内容基础上做精装修——视觉升级、装饰增强、局部内容深化。不推倒重来。

**执行方式**：
  a. 用户原始任务原文（原封不动引用用户的请求）
  b. draft 文件路径（布局骨架）
  c. 该页 content.md 内容（大纲素材，直接内联该页完整章节）
  d. 源文档相关段落（深度素材，直接内联源文档对应章节原文）
  e. 风格参考文档路径
  f. 阶段二规则
  g. 输出文件路径

**执行步骤**：
1. 用 Read 工具读取 page_N_draft.svg（禁止凭记忆）
2. 阅读 prompt 中的大纲和源文档素材，识别可补充的细节信息
3. 将 draft 坐标×2 作为基准定位，在模块内部做精装修

**布局约束**：
- 分区方案原样保留，坐标×2 基准，允许±20%微调
- 元素只增不减，禁止引入新的大面积留白
- 空间变化就近消化

**精装修要求**（在模块内部完成，不改整体布局）：
- 视觉精细化：三级字体体系、精细线条(0.5~1px)、渐变+阴影、每页1-2个视觉锚点
- 图表强化：简单图形升级为精致图表（如加网格线、数据标签、图例）
- 装饰增强：页面边缘/背景层加轻量几何装饰，不挤占内容空间
- 局部内容深化：关键数字加说明注释、结论加摘要高亮、列表拆为精致卡片、增加微图表和状态标识

**禁止**：重划分区、大幅移动模块、删除/合并模块、打散布局、降低信息密度

**验收标准**：
- 布局与 draft 一致，信息密度只增不减
- 模块内部有实质性视觉升级与装饰增强
- 达到"可直接用于正式展示"的完成度

**draft 文件保留以供对比，最终打包 pptx 不含 draft。**

### 打包交付
1. 调用 `scripts/svg_to_pptx/svg_to_pptx.py` 将所有 SVG 转换为 PPTX
2. 调用 `pptx_to_pdf.js` 为 PPTX 生成 PDF
3. 交付最终产物：`.pptx` 和 `.pdf` 文件

## 关键规范

### SVG 页面规格
- **阶段一（draft）**: viewBox="0 0 640 360"，命名 `page_N_draft.svg`
- **阶段二（成品）**: viewBox="0 0 1280 720"，命名 `page_N.svg`

### 内容密度
- 默认推荐：500 字左右/页
- 最大限制：10 页（如无明确要求）

### 风格规范

#### 华为风格（当用户指定"华为风格"时激活）

**必读**：生成 SVG 页面时必须遵循 `huawei.md` 规范，核心要求如下：

**1. 页面结构**
- 顶部装饰线：1px 红色线条 `#c7020e`
- 标题栏：高度 90px，包含红色指示条 (4px×30px)、主标题、英文副标题、页码
- 内容区：左右边距一致，高度约 504px

**2. 配色规范**
- 主色（华为红）：`#c7020e` — 用于强调色、主标题、装饰线、红色导航条
- 辅助灰色系：
  - `#8c8c8c` — 辅助文字、图标
  - `#bfbfbf` — 分割线、标签边框
  - `#d9d9d9` — 卡片边框、表格边框
  - `#f2f2f2` — 背景色、填充
- **禁止使用太淡的灰色文字**：正文必须使用黑色或深色，确保可读性

**3. 字体规范**
- 中文字体：微软雅黑（Microsoft YaHei）
- 英文字体：Arial
- 字号：页面标题 28pt、一级标题 17pt、二级标题 16pt、正文 14pt、辅助 11pt
- 行距：1.5 倍

**4. 组件样式**
- 卡片形状：使用直角，禁止圆角
- 边框粗细：统一 1px
- 数据卡片：白色背景，灰色边框，左侧 4px 红色边框
- 红色标签：红色边框 `#c7020e`，浅红背景 `#fff1ef`
- 灰色标签：灰色边框 `#bfbfbf`，浅灰背景 `#f2f2f2`

**5. 布局原则**
- 观点在标题上：标题必须包含核心观点
- 每页传递观点三点以内
- 内容充实，避免过度留白
- 关键内容标红（使用华为红）

**6. 文件引用**
- 风格规范文件：`huawei.md`（完整规范文档）


## 故障排查

- 如果 SVG 转换为 PPTX 失败，检查 `scripts/svg_to_pptx/svg_to_pptx.py` 的依赖：`pip install python-pptx lxml`
- 如果页面生成质量问题，参考本 SKILL.md 的【阶段二】验收标准
