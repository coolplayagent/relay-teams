OpenAI introduces workspace agents in ChatGPT: shared, Codex-powered agents for teams that can handle complex tasks and long-running workflows within enterprise permissions and controls.

## Main ideas

- workspace agents are an evolution of GPTs
- they run in the cloud and can keep working when the user is away
- they are designed to be shared inside an organization
- they can gather context from connected systems, follow team processes, ask for approvals, and keep work moving across tools

## Product / engineering relevance

The article is relevant to multi-agent engineering because it documents:

- cloud-hosted long-running agents
- organization-scoped permissions and safeguards
- shared agent construction and reuse
- schedule-based and Slack-based execution surfaces
- analytics, compliance visibility, and governance controls
- memory and improvement-through-use patterns
